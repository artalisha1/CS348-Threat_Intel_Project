# AI Usage Disclosure

**Which AI tools were used:**
I used Google Gemini while working on this project.

**What tasks the AI assisted with:**
I wrote most of the core logic, database schema design, and API routing myself. 
I used AI to assist with the following:
0. **Planning & Setup:** I used AI to help figure out the best way to structure my project. For example, the various aspects I should implement for each stage, what filters should be included and the various API routes that should be included. For the database schema design, I used AI to help me figure out the proper relationships between the tables and the various columns that should be included in each table. I also used AI to help me set up the backend, Node.js and the Express server to communicate with MySQL database. Along with this, I used it to help me research and understand various functions and features of Express and Node.js that I was not familiar with. I also asked it to help me figure out the best way to design the various API routes to allow for filtering and sorting of the data.  
1. **Debugging:** Helping me identify errors in my SQL queries and resolving any bugs when communicating between the frontend and the backend server. 
2. **Formatting & Aesthetics:** Generating CSS styling and layout of the webpage to ensure the user interface looked clean. I wanted to focus my efforts on the  database concepts as well as developing the backend, frontend logic and how they interact with each other. However, I still understood all the code that was being generated for the stylist elements. 
3. **Commenting & Documentation:** I wrote my own documentation and comments for the project, but I sometimes asked it to double check my documentation and comments to make sure they were clear and concise.
4. **Code Generation:** I used AI to generate code snippets at times for things I was unfamiliar with or things very specific. For example, generating sample data for the database or creating the connection pool. I also used it to help me create the various API routes (but not the logic within them) and the specific syntax used for this. I asked it to help me determine the most useful indexes to implement and where I could implement transactions. I used it to generate error checking code for the backend server to ensure that it was robust and could handle various error cases. In the frontend, I used it to generate code snippets for various ui elements and components that I was not familiar with the syntax for, but I made sure to modify them to fit my needs and understand all the code generated.
5. **Deployment:** I used AI to help me figure out how to deploy my application to Google Cloud Platform. I asked it to how to set up the backend and database to work together on GCP as well as other commands I would need to use. Along with this, I asked it how to change my code so it can properly deploy on GCP.  

**How the student verified or modified the AI-generated output:**
I was very diligent in verifying any code the AI suggested. For any SQL queries or database logic the AI helped debug, I first ran the queries manually in my MySQL client to verify they returned the expected results and correctly utilized the indexes I designed. I also checked the MySQL documentation (specifically regarding transactions and isolation levels) and the Express.js documentation to ensure the code was accurate and followed best practices. Finally, I heavily modified any AI-generated snippets to match my specific database schema and project requirements rather than blindly copy-pasting code. I ensured I fully understood every single line of code and database design decision implemented in this project.
